# EECS3311 Project

## Overview

This repository contains the UML diagrams created for **Deliverable 1** and **Deliverable 2** of the EECS 3311 group project.  
The project analyzes and models a **Lab Equipment Reservation System** for York University based on the provided client requirements.

## Contents

### Deliverable 2

- Class diagram and sub-diagrams illustrating each of the six design patterns
- Component diagram

### Deliverable 1

- **Use Case Diagrams** – Identify system actors and main functionalities
- **Activity Diagrams** – Show workflows for system scenarios
- **Sequence Diagrams** – Describe interactions between system components
- **Class Diagram** – Initial system design including classes, attributes, and relationships

## System Description

The system allows users (students, faculty, researchers, and guests) to:

- Register and log into the platform
- Reserve laboratory equipment
- Pay deposits for bookings
- Modify or cancel reservations
- Extend reservations if equipment is available

Lab managers manage equipment, while sensors track equipment usage automatically.

## Notes

All diagrams were created based on the project requirements document provided in EECS 3311 Deliverable 1 and Deliverable 2.  
AI tools were used as assistance during modeling and were verified and adapted according to UML standards.
